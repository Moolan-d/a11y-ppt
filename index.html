Web Accessibility 及其相关规范

    帮助视觉、听觉、身体、语音、认知、语言、学习和神经系统残疾的人群更方便的访问 Web 内容。

    提高某些特定场景下的访问便捷性:

    • 人们使用手机、智能手表、智能电视以及其他具有小屏幕、不同输入方式等特点的设备。
    • 年龄增长导致能力变化的老年人
    • 因手臂骨折或眼镜丢失等“暂时性残障”的人群
    • 在强烈阳光下或无法听取音频的环境中等“情境限制”的人群
    • 使用缓慢的互联网连接或带宽有限或昂贵的人群

相关规范
WCAG2.1

Web Content Accessibility Guidelines 可访问性指南

以可感知为例

WCAG针对可访问性是否成功的标准有三个等级 —— A、AA 和 AAA。

对于AA级遵从性，Web页面符合所有A级和AA级成功标准，或提供符合AA级要求的替代版本。
WAI_ARIA

Web Accessibility Initiative - Accessible Rich Internet Applications(可访问的富互联网应用)

WAI-ARIA 是一组属性和角色，用于描述和标记交互式部分，以便于屏幕阅读器和辅助技术处理它们

如何提高页面的可访问性？
  1. 对齐 WCAG 的标准 进行页面元件开发
  2. 利用 WAI-ARIA 构建健壮的 Accessibility Tree
    • 封装或使用具有 Accessibility 能力的组件（如 react-spectrum )
  3. 善用 tabIndex 进行焦点控制
